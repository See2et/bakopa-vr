# プロダクト概要

## プロダクトの目的
BakopaVR は、分散型 Social-VR プラットフォーム「SuteraVR」の Minimal Value Product (MVP) として、P2P ネットワーク基盤が実際のユーザー行動に耐えうるかを検証する試験的プロジェクトです。Unity クライアントと Rust 製ネットワークスタック（iroh）を組み合わせ、中央サーバーに依存しない没入型コミュニティ体験の成立性を最短サイクルで確認します。

## ターゲットユーザー
- SuteraVR コミュニティの初期コントリビューターやモデレーター。
- 中央集権インフラに懐疑的な分散志向クリエイター・イベント主催者。
- リアルタイム通信の新技術を検証したい VR ネットワークエンジニア／研究者。

## 主要機能
1. **マイクロソーシャルスペース**: 最大 6 名が参加できる小規模な共有空間で、アバター姿勢とハンドトラッキングを同期。
2. **P2P 音声チャンネル**: iroh のストリーム機能経由で双方向音声を低遅延伝送し、障害時は近接ピアへ自動フェイルオーバー。
3. **セッションブックマーク**: 招待コードと P2P ノード ID を一体化したシンプルな参加フローで、サーバーレスの出席管理を実現。

## ビジネス目標
- リリース後 4 週間で、P2P セッション成功率 90% 以上の安定性を実証する。
- SuteraVR コミュニティのアーリーサポーター 200 名以上からネットワーク品質フィードバックを収集する。
- 2026 年度までに分散ホスティング機能の有償提供モデル（例: 有料リレーノード）を設計するためのデータを取得する。

## 成功指標
- **セッション成功率**: 接続完了から 15 分継続したセッション比率を 90% 以上に維持。
- **平均往復遅延**: 音声 RTT の中央値 120 ms 以下。
- **継続利用率**: 初回参加者の 50% 以上が 2 回目のセッションに参加。

## プロダクト原則
1. **ピア優先の信頼性**: サーバーに依存せず、ピア間でリレー・リトライを完結できる設計を最優先する。
2. **可観測性の担保**: 分散環境でもネットワーク品質を追跡できる diag/メトリクスを初期段階から埋め込む。
3. **設置容易性**: 1 回のビルドとショートチュートリアルでイベント主催者が独自ノードを立ち上げられるユーザー体験を提供。

## モニタリングと可視化（該当する場合）
- **ダッシュボード種別**: P2P セッションの接続グラフと遅延統計を表示する Web ベース観測ツール。
- **リアルタイム性**: iroh のメトリクス API から 5 秒間隔でイベントをバッチ収集し、Grafana 互換のエンドポイントへ送信。
- **主要メトリクス**: セッション成功率、ノードごとの帯域幅、音声 RTT、障害時リレー回数。
- **共有手段**: JSON エクスポートと read-only ダッシュボードリンクで外部パートナーにも共有可能。

## 将来ビジョン
BakopaVR を足がかりに、SuteraVR 全体での分散コミュニティ運営（大規模イベント、ユーザー生成空間、トークングート等）に耐える P2P 基盤を整備する。

### 発展アイデア
- **リモートアクセス**: モバイルから観戦・モデレーションできる軽量 Web クライアント。
- **分析機能**: 地理分布と時間帯別にピア接続品質を可視化し、最適なリレー戦略を提案するレコメンド。
- **協働機能**: マルチシーン同期や共同編集ボードを備えた分散コラボレーションモード。
